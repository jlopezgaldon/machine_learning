Regresión logística¶

La funcin logstica, curva logstica o curva en forma de S es una funcin matemtica que aparece en diversos modelos de crecimiento de poblaciones, propagacin de enfermedades epidmicas y difusin en redes sociales. Dicha funcin constituye un refinamiento del modelo exponencial para el crecimiento de una magnitud. Modela la funcin sigmoidea de crecimiento de un conjunto P.

Dada $x$ e $y$ calcular $\hat{y}$ siendo $\hat{y} = P(y = 1 | x)$ donde $0 \leq \hat{y} \leq 1$

Para ello, deberemos calcular $w \in \mathbb{R}^n$ y $b \in \mathbb{R}$ donde $\hat{y} = \sigma(w^Tx + b) = \sigma(z) \approx y$ , es decir, $\sigma(z) = \frac{1}{1 + e^{-z}}$

A continuacin, realizaremos un grfico con la funcin logstica:

def reg_log (x):
        '''
        Funcin de clculo regresin logstica o sigmoide.
        Dada x (una secuencia de datos), calcula la sigmoide
        '''
        return 1 / (1 + np.exp(-x))

El concepto es el siguiente:

Si x is muy grande $\sigma(x) \approx 1$, mientras que si x es un nmero muy pequeo $\sigma(z) \approx 0$

Función de pérdida (error)¶

Como $\hat{y} \approx y$, tenemos una prida o error:

La funcin de coste para la regresin logstica es la siguiente:

$$\ell(y, \hat{y}) = -(ylog(\hat{y}) + (1 - y)log(1 - \hat{y})$$

Por qu est funcin?

Si $y = 0$ queremos $\hat{y}$ pequeo, mientras que si $y = 1$ querremos un valor alto para $\hat{y}$. Dicho con otras palabras, queremos penalizar el resultado.

Función de coste¶

La funcin de coste agrega el resultado de la funcin de prdida para cada $(x_{i}, y_{i})$.

Dado n, el tamao de la muestra, la funcin de coste es:

$$\jmath(w, b) = \frac{1}{n}\sum_{1}^{n}\ell(y^{i} \hat{y}^{i}) = -\frac{1}{n}\sum_{1}^{n}((y^{i}log(\hat{y}^{i}) + (y^{i} +1)log(1 - \hat{y}^{i}))$$

donde el gradiente de la funcin de coste es:

$$\frac{\partial J(\theta)}{\partial \theta_j} = \frac{1}{n} \sum_{i=1}^{n} (h_{\theta}(x^{(i)}) - y^{(i)})x_j^{(i)}$$

Recordemos que:

$\hat{y} = h_{\theta} = \sigma(w^Tx + b)$

$\frac{1}{n}$ obtiene el error medio

Una vez tenemos explicada la teora, defineremos una funcin para calcular la funcin de coste:

In [9]:
def fun_coste (theta, x, y):
    '''
    Calculo de la funcin de coste y gradiente de la funcin:
    theta = parmetros del modelo
    x = vector de x
    y = vector de y
    '''
    n = len(y)                                                  # n es el tamao de y, lo usaremos para el -(1/N)
    y_hat = reg_log(np.dot(x , theta))                          # y_hat = sigma(h0 * x)
    error = (y * np.log(y_hat)) + ((1 - y) * np.log(1 - y_hat)) # error = todo lo que est despus del sumatorio
    cost = -1 / n * sum(error)                                  # cost = la aplicacin de toda la frmula -(1/n)*sumatorio(error)
    gradient = 1 / n * np.dot(x.transpose(), (y_hat - y))       # gradient = (y_hat - y) * t(x)
    return cost[0] , gradient

Descenso del gradiente¶

El mtodo del descenso del gradiente es un algoritmo de optimizacin que permite converger hacia el valor mnimo de una funcin mediante un proceso iterativo. En aprendizaje automtico bsicamente se utiliza para minimizar una funcin que mide el error de prediccin del modelo en el conjunto de datos. A esta funcin de error se le suele denominar funcin de coste e identificar con $J(\theta)$, en donde $\theta$ hace referencia a los parmetros del modelo.

Para identificar el mnimo de la funcin el mtodo del descenso del gradiente calcula la derivada parcial respecto a cada parmetro en el punto de evaluacin. La derivada indica el valor y sentido en que se encuentra el mnimo ms prximo. Este puede ser tanto un mnimo local como global, el mtodo no los puede diferenciar. El resultado de la derivada se le resta a cada uno de los parmetros multiplicado por la velocidad de aprendizaje $\alpha$. La velocidad de aprendizaje generalmente tiene un valor entre 0 y 1 e indica lo rpido que converge el algoritmo. Es importante notar que es necesario seleccionar un valor adecuado. Un valor demasiado bajo puede provocar que nunca se alcance el mnimo. Por otro lado, un valor lo demasiado alto podra saltarse el mnimo.

El modo de funcionamiento bsico del mtodo del descenso del gradiente es:

  1. Inicializar los parmetros $\theta$ a un valor de inicio
  2. Indicar la velocidad de aprendizaje del algoritmo $\alpha$
  3. Obtener la derivada de $J$ en el punto $\theta$
  4. Sustraer la derivada por la velocidad de aprendizaje al valor actual del parmetro
  5. Actualizar el valor de $\theta$ el nuevo valor
  6. Comprobar el cambio en la actualizacin de los parmetros es inferior a un fijado previamente (llamada criterio de parada).
  7. En caso afirmativo finalizar la ejecucin, en caso contrario volver al punto 3.

def fun_descenso_gradiente(x, y, theta, alpha, iterations):
    '''
    Clculo del descenso del gradiente.
    x = variables explicativas estandrizadas con trmino independiente
    y = variable explicada
    theta = parmetros iniciales
    alpha = tasa de aprendizaje (tomara valores entre 0 y 1)
    interations = nmero de iteraciones
    '''
    costs = []                                     # Nos definimos el objeto costs en blanco
    for i in range(iterations):                    # Hacemos un loop con las iteraciones hasta que consiga converger utilizando
        cost, gradient = fun_coste(theta, x, y)    # la anterior (theta - = alpha * gradient)
        theta -= (alpha * gradient)
        costs.append(cost)
    return theta, costs
